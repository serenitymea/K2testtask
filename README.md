# Order Management System

Невеликий сервіс для обліку замовлень на FastAPI. Проєкт робився як простий backend без зайвої складності, але з нормальною структурою та базовою бізнес-логікою. Тут можна створювати клієнтів, товари та замовлення, а також автоматично рахувати суму замовлення.

Для бекенду був вибраний FastAPI, тому що він простий у роботі, швидкий і не змушує писати багато зайвого коду. Також у нього одразу є Swagger документація, тому API можна тестувати прямо в браузері без додаткових інструментів.

База даних — PostgreSQL. Вона стабільна, нормально працює з транзакціями та підходить для таких речей як замовлення і гроші. Для роботи з базою використовується SQLAlchemy ORM, щоб працювати через Python-моделі, а не писати SQL вручну. Так код виглядає чистіше і простіше підтримується.

Docker доданий просто для зручності. Щоб не налаштовувати все руками і не ловити проблеми з версіями, проєкт можна підняти однією командою. Для тестів використовується SQLite in-memory, бо це швидше і не треба окремо запускати PostgreSQL.

У замовленні обов’язково має бути клієнт і хоча б один товар. Сума замовлення рахується автоматично, а ціна товару зберігається на момент покупки, щоб старі замовлення не змінювались після оновлення товарів.

## Запуск

```bash
docker compose up --build
```

Після запуску:

* http://localhost:8000
* http://localhost:8000/docs

## Тести

```bash
pytest -v
```
